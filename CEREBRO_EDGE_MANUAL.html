    CEREBRO EDGE 요약 메뉴얼¶

    2026.05.26 기준 Ver 0.11

    1. CEREBRO EDGE¶

    1.1 CEREBRO EDGE란¶

    • 업무 흐름을 시각화하고 실행 구조까지 연결하는 노코드 기반 시스템 구축 플랫폼 입니다.
      방대한 설계 문서를 일괄 입력하면 AI가 이를 분석해 서브시스템 단위의 업무 흐름을 자동 생성하여 시스템 구축 시간을 획기적으로 단축합니다.
      또한 시스템의 소스코드를 자동 분석해 구조, 호출 관계, 데이터 흐름을 파악하여 이를 기반으로 자연어로 시스템을 탐색하고 시스템 구조와 업무 흐름을 쉽게 이해할 수 있습니다.

    1.2 주요 기능¶

    🔀 업무 플로우 디자이너¶

    • 시각적 프로세스 모델링
    • 사용자 친화적 UI 환경
    • 컴포넌트 기반 No-Code 개발
    • 현업-개발자 통합 협업 환경

    🔄 AI 기반 리버스 엔지니어링¶

    • 로직 기반 플로우 자동 변환
    • 비즈니스 규칙(Rule) 식별 및 추출
    • 현행 시스템 자산 분석
    • 코드-플로우 양방향 연계 추적

    🤖 지능형 실행엔진 (LLM 기반)¶

    • LLM 기반 컨텍스트 추론
    • 동적 워크플로우 경로 제어
    • 유연한 모델 및 기능 연계
    • 지능형 데이터 가공 및 결과 도출

    🖥️ 실시간 모니터링¶

    • 엔드투엔드(E2E) 워크플로우 실시간 관제
    • 실시간 시스템 자원 모니터링
    • 장애 추적 및 신속한 원인 분석
    • 상세 오류 로그 추적

    1.3 특장점¶

    ⚡ 빠른 성능¶

    • 실시간 응답성 확보

    2. 로그인¶

    • 이메일 주소, 이름, 전화번호를 입력하여 회원가입을 신청합니다.
    • 가입 완료 후 관리자의 승인을 거쳐 최종 계정이 활성화됩니다.
    • 로그인 완료 시 권한에 따라 메인 화면 또는 비즈니스별 기능이 제어됩니다.

    3. 메인화면¶

    상단 메인 메뉴를 통해 핵심 기능에 빠르게 접근할 수 있으며, 좌측 탐색기와 중앙 비즈니스 영역의 상호 연동 구조를 통해 생성된 비즈니스를 직관적으로 관리합니다.

    상단 메인 메뉴를 통해 핵심 기능에 빠르게 접근할 수 있으며, 좌측 탐색기와 중앙 비즈니스 영역의 상호 연동 구조를 통해 생성된 비즈니스를 직관적으로 관리합니다.

    3.1 메인 메뉴 영역¶

    • 비즈니스, 데이터소스, SQL 쿼리 도구 등 솔루션의 모든 핵심 기능 메뉴로 즉시 이동하여 화면을 전환합니다.
    메뉴
    설명
    비즈니스 워크플로우를 작성하고 관리합니다.
    비즈니스에서 사용할 데이터베이스 연결 정보를 등록하고 관리합니다.
    실시간 쿼리 실행을 통해 결과를 조회하고 비즈니스 로직을 사전에 검증합니다.
    지정된 주기에 맞춰 특정 비즈니스가 자동으로 실행되도록 관리합니다.
    처리 이력을 실시간으로 기록하여 문제 추적과 데이터 보관을 지원합니다.
    비즈니스의 실시간 상태와 성능 통계를 통합 관제합니다.
    ----------------------------------------------

    3.2 탐색기 영역¶

    • 중앙 비즈니스 영역과 실시간으로 연동되는 구조이며, 전체 비즈니스 계층을 트리 형태로 파악하고 원하는 경로로 빠르게 이동합니다.

    3.3 비즈니스 영역¶

    • 좌측 탐색기와 실시간으로 연동되는 구조이며, 하위 항목들을 카드나 테이블 형태로 표출하여 신규 생성 및 원하는 비즈니스로 진입합니다.

    4. 비즈니스¶

    4.1 비즈니스 생성¶

    • 비즈니스 생성 버튼을 클릭하여 새로운 비즈니스를 생성합니다.
    • 빈 쿼크플로우, AI생성, JSON 가져오기 생성, 소스코드를 업로드하여 일괄 비즈니스를 생성할 수 있습니다.

    4.1.1 빈 워크플로우 생성¶

    가장 기초적인 형태의 워크플로우를 생성합니다. 처음에는 [시작]과 [종료] 블록만 존재하며, 사용자가 원하는 블록을 자유롭게 배치하고 연결하여 비즈니스를 작성할 수 있습니다.

    • 비즈니스 이름과 설명을 입력한 후 [생성] 버튼을 클릭합니다.
    • [시작]과 [종료] 블록만 있는 디자이너 화면으로 전환되며, 필요한 블록들을 추가하고 연결하여 비즈니스를 완성합니다.

    4.1.2 AI로 생성¶

    사용자가 입력한 자연어 설명을 기반으로 AI가 비즈니스 로직을 분석하여, 이에 맞는 워크플로우를 자동으로 구성해 주는 생성 방식입니다.

    • 워크플로우 설명을 입력하고 [AI로 생성] 버튼을 클릭합니다.
    • 더 정확한 분석을 위해 문서(PDF) 또는 소스코드 파일(Java, Python, JavaScript 등)을 추가로 첨부할 수 있습니다.
    • AI가 입력된 설명에 부합하는 블록들과 적절한 로직 흐름을 자동으로 생성하여 디자이너 화면에 배치합니다.

    4.1.3 JSON 가져오기¶

    JSON 파일을 불러와 워크플로우를 생성합니다. 기존 비즈니스를 그대로 복원하거나, 이를 기반으로 추가적인 편집을 하고 싶을 때 유용하게 사용할 수 있습니다.

    • JSON 파일을 선택하고, 새로 만들 비즈니스의 이름과 설명을 입력합니다.
    • 비즈니스를 관리할 폴더 분류를 선택합니다.
    • 모든 입력이 완료되면 [가져오기] 버튼을 클릭하여 비즈니스를 생성합니다.

    4.1.4 일괄 생성¶

    기존 프로그램의 소스코드를 시스템이 스스로 읽고 분석하여, 복잡한 비즈니스 메뉴(트리)와 업무 흐름도(워크플로우)를 한 번에 자동으로 그려내는 편리한 생성 방식입니다.

    • 분석할 소스코드를 다음 방식 중 하나를 선택하여 지정합니다.
      • 파일 업로드: 압축파일(.zip 등) 또는 소스폴더 직접 업로드
      • 저장소 연결: Git 또는 SVN 저장소 URL 및 계정 연동
      • 경로 지정: 소스코드가 위치한 특정 절대 경로 입력
    • 소스 지정이 완료되면 [업로드 & 분석 시작] 버튼을 클릭합니다.
    • 시스템이 코드를 분석하여 비즈니스 트리와 해당하는 워크플로우들을 자동으로 일괄 생성합니다.

    4.2 비즈니스 수정¶

    • 비즈니스 영역의 각 카드나 목록 우측에 있는 [더보기(세로 점 3개)] 아이콘을 클릭합니다.
    • 팝업 메뉴에서 [수정] 을 선택합니다.
    • 비즈니스 수정 모달 창에서 기존의 이름과 설명을 원하는 대로 변경한 후 저장합니다.

    4.3 비즈니스 내보내기¶

    • 비즈니스 영역의 각 카드나 목록 우측에 있는 [더보기(세로 점 3개)] 아이콘을 클릭합니다.
    • 팝업 메뉴에서 [내보내기] 를 선택합니다.
    • 해당 비즈니스의 전체 워크플로우 구성 정보가 담긴 JSON 파일이 즉시 다운로드됩니다.

    4.4 비즈니스 폴더이동¶

    • 비즈니스 영역의 각 카드나 목록 우측에 있는 [더보기(세로 점 3개)] 아이콘을 클릭합니다.
    • 팝업 메뉴에서 [폴더로 이동] 을 선택합니다.
    • 이동할 대상 폴더를 지정하면 해당 비즈니스가 선택한 위치의 폴더로 즉시 이동합니다.

    4.5 디자이너¶

    4.5.1 워크플로우 작성¶

    화면에 추가한 블록들을 서로 연결하여 하나의 완성된 비즈니스 규칙을 디자인하는 과정입니다. 마우스 드래그를 통해 블록 간의 실행 순서를 정의할 수 있습니다.

    • 새 비즈니스를 만들고 디자이너 화면에 들어오면, 기본 뼈대인 [시작] 블록과 [종료] 블록이 자동으로 생성되어 있습니다.
    • 두 블록 사이에 필요한 데이터 처리 블록(DB, LLM 등)과 흐름 제어 블록(IF, 반복 등)을 자유롭게 추가합니다.
    • 각 블록의 연결점(포트)을 마우스로 클릭한 채 다음 블록의 연결점까지 드래그하여 선으로 이어줍니다.
    • IF-THEN이나 IF-ELSE 같은 조건 블록은 참(True)과 거짓(False) 경로에 따라 각각 다른 블록으로 선을 연결합니다.
    • [시작] 블록부터 내가 추가한 블록들을 거쳐 [종료] 블록까지 모든 흐름선이 끊김 없이 연결되면 비즈니스가 완성됩니다.
    • 잘못 연결된 선은 선을 클릭한 후 Delete 키를 누르거나, 마우스 우클릭 메뉴를 통해 간편하게 지우고 다시 연결할 수 있습니다.

    4.5.2 블록 추가¶

    블록은 워크플로우를 구성하는 가장 기본적인 단위입니다. 디자이너 화면에서 마우스 우클릭을 통해 필요한 블록을 자유롭게 추가하고 배치할 수 있습니다.

    • 디자이너 캔버스 영역의 빈 곳에서 마우스 우클릭을 합니다.
    • 콘텍스트 메뉴(팝업 메뉴)가 나타나면 원하는 블록을 선택하여 화면에 추가합니다.
    • 제공되는 블록은 크게 데이터 처리를 담당하는 블록과 흐름 제어를 담당하는 블록으로 나뉩니다.

    4.5.3 메모 추가¶

    복잡한 비즈니스 로직 사이에 설명이나 주석을 남길 수 있는 기능입니다. 협업자와의 소통을 돕고, 추후 워크플로우를 유지보수할 때 로직을 쉽게 이해할 수 있도록 지원합니다.

    • 디자이너 캔버스 영역의 빈 곳에서 마우스 우클릭을 합니다.
    • 콘텍스트 메뉴(팝업 메뉴)가 나타나면 [메모 추가] 를 선택하면 화면에 메모 상자가 생성됩니다.
    • 작성이 완료된 메모를 안전하게 보호할 수 있는 잠금 기능을 제공합니다.

    4.5.4 워크플로우 실행¶

    작성한 비즈니스 규칙을 실제로 동작시켜 로직이 의도한 대로 잘 흘러가는지 결과를 확인할 수 있습니다.

    • 새 비즈니스를 만들고 디자이너 화면에 들어오면, 기본 뼈대인 [시작] 블록과 [종료] 블록이 자동으로 생성되어 있습니다.
    • 해당 화면에서 비즈니스 로직 테스트에 필요한 변수값들을 입력한 후, 하단의 [실행] 버튼을 다시 한번 클릭하면 워크플로우가 동작합니다.
    • 실행 중에는 현재 단계, 성공, 오류 등의 진행 현황이 각 블록의 색상으로 실시간 표기되어 로직의 흐름과 상태를 직관적으로 확인할 수 있습니다.

    4.5.5 워크플로우 내보내기¶

    작성한 워크플로우를 사용자가 원하는 소스코드나 문서(산출물) 형태로 변환해 다운로드하는 기능입니다.

    • 시스템에 탑재된 AI 모델이 워크플로우의 블록 구조와 세부 설정 등을 정밀하게 분석합니다.
    • 분석을 바탕으로 기계적인 코드가 아닌, 사람이 직접 읽고 이해할 수 있는 자연스러운 코드나 깔끔한 문서 형태로 출력해 줍니다.
    • 소스코드: Java, Python, JavaScript 등 실제 개발 프로젝트에 바로 적용할 수 있는 코드 형태
    • 산출물 (문서): Markdown, HTML, PDF 등 보고서나 개발 기획서로 활용할 수 있는 문서 형태

    4.5.6 워크플로우 버전 기록¶

    비즈니스의 변경 이력을 관리하는 기능입니다. 과거 특정 시점의 버전을 확인하고, 해당 상태를 불러와 다시 편집할 수 있습니다.

    • 디자이너 화면 상단의 [버전 기록] 버튼을 클릭하여 지금까지 저장된 버전이력 목록을 확인합니다.
    • 목록에서 원하는 버전을 클릭하면 해당 시점의 워크플로우 화면으로 전환되며, 그 상태에서 이어서 편집을 진행할 수 있습니다.

    4.5.9 워크플로우 찾아바꾸기¶

    워크플로우 내에 배치된 수많은 블록 중 특정 조건에 맞는 블록을 빠르게 검색하고, 해당 블록의 내용을 화면에서 바로 수정할 수 있는 기능입니다.

    • 디자이너 화면의 [찾아바꾸기] 버튼을 클릭하여 모달(팝업) 화면을 띄웁니다.
    • 찾고자 하는 블록의 조건을 입력하여 원하는 블록들을 조회합니다.
    • 검색 결과 목록에서 해당 블록을 선택하여 내용을 즉시 수정할 수 있습니다.
    • 규모가 큰 워크플로우에서 마우스 스크롤 없이 미니 플로우 활성화 위치를 보고 원하는 블록을 정확하게 찾아 수정 가능합니다.

    4.5.10 워크플로우 정렬¶

    디자이너 위에 배치된 블록들을 원하는 기준에 맞춰 깔끔하게 정렬하는 기능입니다. 복잡해진 워크플로우의 가독성을 높이고 흐름을 한눈에 파악할 수 있도록 도와줍니다.

    • 디자이너 화면 상단의 [정렬] 버튼을 클릭하여 원하는 정렬 기준을 선택합니다.
    • 수평 정렬: 왼쪽 맞춤, 가운데 맞춤, 오른쪽 맞춤
    • 수직 정렬: 위쪽 맞춤, 중간 맞춤, 아래쪽 맞춤

    5. 데이터소스¶

    비즈니스 워크플로우에서 사용할 수 있는 데이터베이스 연결정보를 관리합니다.

    • 데이터소스를 한 번 등록하면 이후 모든 DB 블록에서 드롭다운으로 선택해 사용할 수 있습니다.
    • 데이터베이스는 Oracle, MySQL, MariaDB, PostgreSQL 타입을 지원합니다.
    • 커넥션 풀 크기, 최소 유휴 커넥션 수 등 안정적인 DB 운용을 위한 세부 설정을 관리합니다.
    • 등록된 데이터소스가 현재 어떤 비즈니스 워크플로우와 DB 블록에서 참조되어 사용 중인지 한눈에 확인하고 추적할 수 있습니다.

    5.1 데이터소스 등록¶

    • DB 블록에서 사용할 데이터베이스 연결 정보를 등록합니다.
      • 데이터소스 추가하기 버튼을 클릭합니다.
      • 데이터소스 이름, 데이터베이스 타입, 호스트, 포트, 데이터베이스 이름, 사용자명, 비밀번호를 입력합니다.
      • 연결 테스트를 통해 정상 접속여부를 확인합니다.
      • 생성 버튼을 클릭하여 데이터소스를 추가합니다.

    5.2 데이터소스 사용현황¶

    • DB 블록에서 현재 사용중인 데이터 블록을 확인합니다.
      • 데이터 소스 내 사용현황에서 상세보기를 클릭합니다.
      • 해당 DB에서 사용중인 블록 정보를 확인할 수 있습니다.

    6. SQL 쿼리도구¶

    등록된 데이터소스에 직접 SQL 쿼리를 실행하여 결과를 실시간으로 조회하고, 워크플로우 적용 전에 로직을 검증하는 도구입니다.

    • 시스템 부하를 방지하기 위해 실시간 실행 결과는 최대 500건까지만 제한합니다.
    • 모든 쿼리 실행 이력을 자동으로 저장하며, 히스토리에서 선택하여 언제든지 재사용할 수 있습니다.
    • 자주 사용하는 주요 쿼리는 별표 아이콘을 클릭하여 간편하게 즐겨찾기로 등록하고 해제합니다.
    • 불필요해진 과거 쿼리 이력은 휴지통 아이콘을 클릭하여 히스토리 목록에서 즉시 제거합니다.

    6.1 쿼리 이력 조회¶

    • 자주 사용하거나 최근에 실행한 쿼리 이력을 조회하고 재실행할 수 있습니다.
      • 즐겨찾기에서 조회할 쿼리를 확인한 후 실행 버튼을 클릭하여 저장된 쿼리 및 결과를 확인할 수 있습니다.
      • 최근실행에서 조회할 쿼리를 확인한 후 실행 버튼을 클릭하여 최근 사용한 쿼리 및 결과를 확인할 수 있습니다.

    6.2 쿼리 직접 실행¶

    • 연결된 데이터베이스에 쿼리를 직접 입력하여 실행할 수 있습니다.
      • 초기화 버튼을 눌러 이전 조회 기록을 삭제합니다.
      • 이후 사용할 쿼리를 입력한 후 실행 버튼을 클릭하여 조회 결과를 확인할 수 있습니다.

    7. 스케줄러¶

    정해진 주기에 따라 특정 비즈니스 워크플로우가 자동으로 실행되도록 예약하고 관리하는 도구입니다.

    - 이름, 비즈니스, 실행 주기(CronTab)를 설정하여 원하는 시점에 비즈니스를 자동 실행합니다.
- 토글 버튼 스위치 조작만으로 스케줄의 사용 여부를 실시간으로 제어합니다.

    7.1 스케줄 생성¶

    • 작업할 스케줄러를 새로 등록합니다.
      • 스케줄러 화면에서 첫 번째 스케줄 추가하기를 클릭합니다.
      • 스케줄 이름, 설명, 사용할 비즈니스, 스케줄을 설정 한 후 생성을 클릭합니다.
      • 이후 생성된 스케줄을 클릭하여 상세 정보를 확인할 수 있습니다.

    7.2 스케줄 수정¶

    • 등록한 스케줄러에 대하여 수정합니다.
      • 스케줄러 화면에서 [...] 버튼 -> 수정 버튼을 클릭하여 등록한 스케줄을 클릭합니다.
      • 수정하고자 하는 항목을 선택하여 수정한 후 수정 버튼을 클릭합니다.
      • 메인 화면에서 변경된 스케줄에 대해 확인할 수 있습니다.

    7.3 스케줄 활성화¶

    • 활성화 시킬 스케줄을 확인하여 활성화합니다.
      • 스케줄러 화면에서 활성화 상태 토글을 클릭하여 스케줄을 활성/비활성화 합니다.
      • 이후 스케줄을 선택하여 활성화 상태를 확인할 수 있습니다.

    7.4 스케줄 삭제¶

    • 불필요한 스케줄을 삭제합니다.
      • 스케줄러 화면에서 [...] 버튼 -> 삭제 버튼을 클릭하여 등록한 스케줄을 클릭합니다.
      • 메인 화면에서 삭제된 스케줄에 대해 확인할 수 있습니다.

    7.5 스케줄 검색¶

    • 등록한 스케줄에 대하여 검색합니다.
      • 스케줄러 화면에서 상단 검색에서 찾고자 하는 스케줄 명을 검색합니다.
      • 이후 메인 화면에서 검색된 스케줄을 확인할 수 있습니다.

    7.6 스케줄 목록보기 형태 변경¶

    • 메인화면의 스케줄 목록 레이아웃을 변경합니다.
      • 스케줄러 화면에서 검색 버튼 옆 테이블보기를 클릭합니다.
      • 테이블 형식으로 스케줄 활성화 및 신규 스케줄을 작성할 수 있습니다.
      • 이후 스케줄러 화면에서 검색 버튼 옆 카드보기를 클릭합니다.

    8. 로그¶

    비즈니스 워크플로우의 시작부터 완료까지의 모든 처리 이력을 기록하고, 문제 추적 및 데이터 보관을 지원하는 도구입니다.

    • 워크플로우의 시작과 완료는 물론, 내부의 개별 블록 단위별 수행 시점과 진행 상황을 상세히 기록하고 관리합니다.
    • 고급 검색 추가 필터를 활용하여 특정 비즈니스의 오류나 데이터 이상 현상을 신속하게 분석합니다.
    • 원하는 로그를 클릭하여 가독성이 높은 테이블 형태나 표준 JSON 포맷으로 세부 내용을 확인합니다.
    • 누적된 로그를 압축하여 디스크 공간을 절약하며, 일정에 따른 자동 압축과 즉시 수동 압축을 모두 지원합니다.

    9. 모니터링¶

    전체 비즈니스의 실행 상태와 성능 통계를 한 화면에서 직관적으로 관제하는 통합 대시보드입니다.

    • 각 비즈니스의 호출 경로(테스트,API,스케줄러 등)와 실행 결과(실행중,완료,실패)를 실시간으로 파악합니다.
    • 전체 프로세스 총 소요 시간은 물론, 내부 블록별 처리 시간을 제공하여 성능 병목 지점을 확인합니다.
    • 개별 이력의 구체적인 실행 내용과 데이터 처리 결과를 대시보드에서 즉시 점검합니다.

    9.1 비즈니스 모니터링¶

    • 각 비즈니스의 호출 경로(테스트,API,스케줄러 등)와 실행 결과(실행중,완료,실패)를 실시간으로 파악합니다.

      • 모니터링 메뉴를 클릭하여 비즈니스 별 실행 현황을 위젯으로 확인합니다.
      • 새로고침 버튼을 클릭하여 실시간 비즈니스 현황을 갱신합니다.
      • 기간, 실행 타입, 상태, 정렬을 선택하여 조건에 맞는 비즈니스 목록을 확인합니다.
      • 상태를 클릭하여 소요시간, 블록별 처리시간을 확인합니다.
      • 워크플로우 확인을 클릭하여 구체적인 실행 내역 및 처리결과 내역을 확인합니다.
